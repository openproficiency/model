## Experience Source

An **Experience Source** is an activity that provides increased proficiency for one or more [topics](./topic.md).

There are typically 2 categories of experience:

- Evidence based - using real-world practical results to understand what what a user already knows
- Learning based - completing trainings to extend what a user knows

### Evidence based experience

[Topics](topic.md) are assigned [scores](topic-score.md) if pre-determined evidence is witnessed in a user's telemetry of a product or service.

Examples:

- A user doing their regular everyday job
- A user practicing on their own to learn new tools
- A user exploring a different domain of work
- A user engaging with hobbies
- A user following a practical tutorial. Since it engages with the real product, it produces usage telemetry.

## Learning based experience

[Topics](topic.md) are assigned [scores](topic-score.md) if an educational organization has performed their own assessments and are confident of a user's proficiency.

Examples:

- Completing an instructor-lead course or tutorial
- Watching an education video
- Passing a quiz, test, or exam

### Avoiding duplicate work

It is recommended to map learning materials to [topics](topic.md). This provides 2 main advantages:

1. It is simple to determine if a user already has proficiency by comparing their [topic scores](topic-score.md) to the learning material's assigned [topics](topic.md).
   - If a learner transfers from one institution to another (and the exercises are assigned topics in both systems), the learner does not need to repeat the exercises.

2. The institution can easily produce a coverage map of their learning offerings by comparing it to the [topic lists](topic-list.md) they intend to teach from.
   - Learning materials can be add, refreshed, or reorganized without invalidating a learner's progress.
